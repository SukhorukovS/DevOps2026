### 2 задание

Во многих системах эти переменные доступны всем, у кого есть права на чтение репозитория (включая форки).
Журналы доступа к переменным обычно не столь детальны и защищены, как для специализированных систем (HashiCorp Vault, AWS Secrets Manager).
Переменные репозитория часто подставляются в команды и могут случайно оказаться в логах сборки, особенно при set -x или при отладке.
Нет встроенного механизма автоматической смены секретов, поэтому старые ключи могут висеть годами.
Один секрет в переменной репозитория становится доступным всем пайплайнам этого репозитория, что увеличивает поверхность атаки.
Нельзя легко настроить динамические политики доступа (например, доступ только для конкретных этапов пайплайна или IP-адресов).
