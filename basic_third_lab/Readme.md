### Сложности при настройке

1. Развернул сервер в докере, и открыл только порт 80, понял что нужно еще 443 порт для ssl подключения только позже, пришлось перезапускать контейнер, хотя уже определенные настройки сделал
2. После того как открыл 443 контейнер не мог понять почему сервер не направляет запрос на указанный файл, оказалось что есть два файла, два конфига, я так понимаю что может быть и больше, и нужно
все указывать в файле default.conf
3. Когда с настройками разобрался некоторое время не мог понять почему отдает 403 forbidden - нужно было выдать права пользователю nginx на папку, где находится сайт
4. Права выданы, что дальше? Правильно! Не указан индексовый файл index index.html index.htm;
5. Ну еще для того чтобы можно было открыть под необходимым именем сайт, нужно прописать домен в hosts
6. Для https выпустил самоподписанный сертификат, но он конечно не хочет работать как нужно
7. Еще понял, когда указываем alias слеши должны быть и в названии и в пути к файлу иначе это может привести к уязвимости path traversal
